## ⚠️ 重要：报告保存策略

**[Request interrupted by user]** - 认证生成完成后，**AskUserQuestion 询问用户保存文档的流程并不能稳定触发**。

### 稳定保存要求

1. **先生成内容，立即保存到桌面**（使用 Write 工具）
2. **再向用户展示结果**，告知保存路径
3. **最后可选地询问**是否需要保存到其他位置（AskUserQuestion 作为增强功能）

### 为什么必须这样设计

- AskUserQuestion 在某些情况下会失败或被打断
- 用户可能看不到选择框，导致报告丢失
- 必须先确保报告已持久化到磁盘，再与用户交互

### 推荐流程

```
1. 完成所有检查 → 生成报告内容
2. 使用 Write 工具 → 保存到 ~/Desktop/BSS-认证-{skill-name}-{评级}-报告.md
3. 向用户展示 → 报告摘要 + 保存路径
4. 可选 AskUserQuestion → 询问是否需要其他位置
   ├─ 成功 → 按用户选择复制/移动到指定位置
   └─ 失败 → 报告已在桌面，用户无损失
```

## 报告撰写指南

### 1. 评级结果部分
- 给出明确的评级（S+/S/A/B/C/D）
- 用一句话概括评价
- 说明主要理由

### 2. 检查依据部分
- 列出所有通过的检查项（✅）
- 列出需要注意的事项（⚠️）
- 列出发现的问题（❌）

### 3. 详细检查部分
- 每个维度都要给出状态（通过/警告/危险）
- 发现的问题要具体说明
- 如果有缓解措施，要明确列出

### 4. 来源可信度部分（T1/T2 适用）
- 说明哪些检查项因来源可信而放宽
- 用表格形式展示放宽政策

### 5. 使用建议部分
- 明确推荐的使用场景
- 提供具体的安全使用指南
- 指出不适用场景
- 提供改进建议（B 级及以下）

### 6. 对比表格（可选）
- 与其他 skill 进行评级对比
- 帮助用户理解相对风险水平
