# BSS 安全认证检查清单

## 来源分级检查

### T1 - 官方/顶级来源
- 来自知名大型技术公司官方渠道
- 来自顶级开源基金会官方仓库
- 有有效的代码签名证书
- 项目维护活跃，有专业安全团队

**示例**：Google, Microsoft, OpenAI, Anthropic, Meta, AWS, Apache, Linux 基金会

### T2 - 可信组织来源
- 来自有实名认证的组织
- 项目有明确的维护团队和治理结构
- GitHub/GitLab 组织账号（非个人账号）
- 项目 Stars > 1000 或有良好声誉
- 有安全响应流程

### T3 - 社区/个人来源
- 个人开发者账号
- 小型社区项目
- 来源无法明确验证
- 新项目或维护不活跃项目

## 危险代码模式检测

### D级触发项（一票否决）
- [ ] 检测 `rm -rf /` 或类似系统破坏性命令
- [ ] 检测 `format`、`fdisk` 等磁盘操作命令
- [ ] 检测 `mkfs`、`dd` 等低级别存储操作
- [ ] 检测删除系统关键目录的命令（/bin, /sbin, /usr, /etc 等）
- [ ] 检测 `eval()` 执行不可信网络代码
- [ ] 检测 `exec()`、`system()` 执行未过滤的用户输入
- [ ] 检测 SQL 注入漏洞（字符串拼接 SQL）
- [ ] 检测 XSS 漏洞（未过滤的输出到 HTML）
- [ ] 检测敏感信息硬编码（密码、密钥、Token）

### C级触发项
- [ ] 检测 `rm`、`del` 删除用户文件操作
- [ ] 检测 `chmod`、`chown` 修改权限操作
- [ ] 检测网络请求发送到第三方域名
- [ ] 检测读取敏感文件（.ssh、.env、config 等）
- [ ] 检测使用 `http` 而非 `https` 传输敏感数据
- [ ] 检测使用不安全的随机数生成器
- [ ] 检测路径遍历漏洞（`../`）
- [ ] 检测不安全的反序列化操作

## 依赖安全检查

### D级触发项
- [ ] 依赖库存在于已知恶意库清单
- [ ] 从不可信源下载可执行代码
- [ ] **T3 来源加载未经验证的动态代码**

### C级触发项
- [ ] 依赖存在高危 CVE 漏洞
- [ ] 依赖来源不可追溯（非官方仓库）
- [ ] 依赖使用通配符版本（可能自动更新）
- [ ] **T2 来源加载动态代码且无来源验证**

### B级检查项
- [ ] 依赖存在中危 CVE 漏洞
- [ ] 依赖已长期未维护（>2年无更新）
- [ ] **T3 来源加载动态代码（即使来自知名 CDN）**

## 数据隐私检查

### D级触发项
- [ ] 未经用户确认上传本地文件到远程（**T3 来源**）
- [ ] 静默收集密码、密钥等敏感信息
- [ ] 将敏感数据传输到未加密通道

### C级触发项
- [ ] **T2/T3 来源**：收集的数据超出功能说明范围
- [ ] **T2/T3 来源**：未明确告知用户数据使用情况
- [ ] 缺乏用户数据删除机制

## 执行安全检测

### D级触发项
- [ ] 无确认直接执行系统级危险操作
- [ ] 以 root/管理员权限执行非必要操作

### C级触发项
- [ ] 危险操作缺乏二次确认
- [ ] 关键操作无回滚机制

## 输入验证检查

### D级触发项
- [ ] 用户输入直接拼接到命令执行
- [ ] 用户输入直接拼接到 SQL 查询
- [ ] 文件路径直接使用用户输入

### C级触发项
- [ ] 缺乏基本的输入长度检查
- [ ] 缺乏输入类型验证

## 报告生成与保存流程检查

### ⚠️ 重要：AskUserQuestion 稳定性问题

**[Request interrupted by user]** - 认证生成完成后，AskUserQuestion 询问用户保存文档的流程**并不能稳定触发**。

#### 问题描述
- AskUserQuestion 工具在某些情况下可能无法正常弹出或被打断
- 用户可能无法看到保存位置的选择提示
- 这会导致报告保存流程中断，用户体验受损

#### 解决方案（必须执行）
- [ ] **生成报告后，首先尝试直接保存到桌面**作为默认行为
- [ ] **同时准备 AskUserQuestion 作为备选**，让用户可以选择其他保存位置
- [ ] **如果 AskUserQuestion 失败或被打断**，确保报告已被保存到默认位置（桌面）
- [ ] **在输出中明确告知用户**报告的默认保存位置和文件名
- [ ] **提供备选指令**，让用户知道如何手动保存（例如："如果未弹出选择框，报告已保存到桌面"）

#### 推荐实现顺序
1. 生成报告内容
2. **立即保存到默认位置**（桌面：`~/Desktop/BSS-认证-{skill-name}-{评级}-报告.md`）
3. 向用户展示报告摘要和保存路径
4. **可选**：尝试询问用户是否需要保存到其他位置（使用 AskUserQuestion）
5. 如果 AskUserQuestion 成功，按用户选择重新保存；如果失败，报告已在桌面

## 评级判定速查

| 发现问题 | 最高评级 |
|----------|----------|
| 任何 D 级触发项 | **D 级** |
| 任何 C 级触发项 | **C 级** |
| 无 C/D 级问题，但未达到 A 级要求 | **B 级** |
| 满足所有 A 级要求 | **A 级** |
| 满足所有 A 级 + S 级额外要求 | **S 级** |
| 满足 S 级 + 通过人工验证 | **S+ 级** |
